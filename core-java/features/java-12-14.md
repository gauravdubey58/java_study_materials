## 1. Switch Expressions (Preview) — JEP 325

### The Problem with Old Switch

```java
// Old switch statement — multiple problems:
// 1. Fall-through by default (need break everywhere)
// 2. Can't return a value
// 3. Verbose for simple value mapping

// BUG — forgot break after MONDAY
String type;
switch (day) {
    case MONDAY:
    case TUESDAY:
        type = "Weekday";  // Falls through to WEDNESDAY case accidentally!
    case WEDNESDAY:
        type = "Midweek";
        break;
    default:
        type = "Other";
}
```

### New Switch Expression — Arrow Form

```java
// Java 14 final form (same syntax introduced as preview in Java 12)

// Switch EXPRESSION — returns a value, no fall-through, no break needed
String type = switch (day) {
    case MONDAY, TUESDAY, THURSDAY, FRIDAY -> "Weekday";
    case WEDNESDAY                         -> "Midweek";
    case SATURDAY, SUNDAY                  -> "Weekend";
};

// Switch expression with block body — use yield to return value
int numLetters = switch (day) {
    case MONDAY, FRIDAY, SUNDAY -> 6;
    case TUESDAY                -> 7;
    case THURSDAY, SATURDAY     -> 8;
    case WEDNESDAY              -> {
        System.out.println("Computing Wednesday...");
        yield 9;  // 'yield' returns value from a block in switch expression
    }
};
```

### Real-World: Order Status Handler

```java
public class OrderStatusHandler {

    // Clean value mapping — replaces if-else chains and old switch
    public String getStatusMessage(OrderStatus status) {
        return switch (status) {
            case PENDING   -> "Your order is being reviewed";
            case CONFIRMED -> "Order confirmed — preparing for dispatch";
            case SHIPPED   -> "Your order is on its way";
            case DELIVERED -> "Order delivered successfully";
            case CANCELLED -> "Order has been cancelled";
            case REFUNDED  -> "Refund processed to your account";
        };
        // Compiler enforces exhaustiveness for enums — no default needed!
        // If a new status is added to enum, this FAILS TO COMPILE — safe!
    }

    // Switch expression for complex logic with yield
    public BigDecimal calculateDiscount(CustomerTier tier, BigDecimal orderAmount) {
        return switch (tier) {
            case BRONZE -> orderAmount.multiply(new BigDecimal("0.05"));
            case SILVER -> orderAmount.multiply(new BigDecimal("0.10"));
            case GOLD   -> orderAmount.multiply(new BigDecimal("0.15"));
            case PLATINUM -> {
                // Complex logic in block — use yield
                BigDecimal base = orderAmount.multiply(new BigDecimal("0.20"));
                BigDecimal bonus = orderAmount.compareTo(new BigDecimal("1000")) > 0
                    ? new BigDecimal("50") : BigDecimal.ZERO;
                yield base.add(bonus);
            }
        };
    }

    // Switch expression can be used inline
    public void processOrder(Order order) {
        log.info("Processing {} order: {}",
            switch (order.getPriority()) {
                case HIGH   -> "🔴 URGENT";
                case MEDIUM -> "🟡 NORMAL";
                case LOW    -> "🟢 STANDARD";
            },
            order.getId()
        );
    }
}
```

### Old Switch Statement Still Works (for side effects)

```java
// Switch STATEMENT (old style) — still valid when you need fall-through
// or when you don't need a return value
switch (command) {
    case "start":
        engine.start();
        metrics.increment("starts");
        break;
    case "stop":
        engine.stop();
        break;
    default:
        log.warn("Unknown command: {}", command);
}
```

### Corner Cases and Gotchas

```java
// 1. yield vs return — yield is for switch blocks, return is for methods
int result = switch (x) {
    case 1 -> 10;               // No yield needed for arrow form
    case 2 -> {
        int v = compute();
        yield v * 2;             // yield required in block form
        // return v * 2;         // COMPILE ERROR — use yield, not return
    }
    default -> 0;
};

// 2. Switch expression must be exhaustive — all cases covered
// For enums: must cover all enum values OR have default
// For int/String: must have default
String label = switch (priority) {   // priority is enum Priority {HIGH, LOW}
    case HIGH -> "urgent";
    // COMPILE ERROR if LOW case is missing and no default
    case LOW  -> "normal";
};

// 3. No fall-through in arrow switch — each case is independent
switch (status) {
    case A -> action1(); // Executes ONLY action1
    case B -> action2(); // No fall-through to action3
    case C -> action3();
}

// 4. Mixing arrow and colon forms is not allowed
switch (x) {
    case 1 -> doA();
    case 2: doB(); break; // COMPILE ERROR — can't mix forms
}
```

## 2. Text Blocks (Preview) — JEP 355

Text blocks allow multi-line strings without escaping. Final in Java 15.

```java
// Before text blocks — escape hell
String json = "{\n" +
              "  \"orderId\": \"" + orderId + "\",\n" +
              "  \"status\": \"PENDING\",\n" +
              "  \"amount\": " + amount + "\n" +
              "}";

String sql = "SELECT c.name, o.total\n" +
             "FROM customers c\n" +
             "JOIN orders o ON c.id = o.customer_id\n" +
             "WHERE o.status = 'PENDING'\n" +
             "ORDER BY o.created_at DESC";

// With text blocks — clean and readable
String json = """
        {
          "orderId": "%s",
          "status": "PENDING",
          "amount": %s
        }
        """.formatted(orderId, amount);

String sql = """
        SELECT c.name, o.total
        FROM customers c
        JOIN orders o ON c.id = o.customer_id
        WHERE o.status = 'PENDING'
        ORDER BY o.created_at DESC
        """;
```

### Text Block Indentation Rules

```java
// The closing """ determines the indentation baseline
// Leading whitespace equal to the closing """ position is stripped

String a = """
        Hello
        World
        """;
// Result: "Hello\nWorld\n"
// (8 spaces stripped from each line; trailing newline included)

String b = """
        Hello
        World
"""; // Closing """ at column 0 — no stripping
// Result: "        Hello\n        World\n"

// Trailing spaces — stripped by default
// Use \s to preserve a trailing space explicitly:
String c = """
        line with trailing space  \s
        next line
        """;
```

### Corner Cases

```java
// 1. Text blocks always end with \n if closing """ is on its own line
String s = """
        hello
        """;
System.out.println(s.length()); // 6 — "hello\n" (includes newline)

// To avoid trailing newline, put closing """ on same line as last content
String s2 = """
        hello"""; // No trailing newline

// 2. Embedded quotes don't need escaping (mostly)
String json = """
        {"key": "value", "nested": {"a": 1}}
        """; // No escaping needed for "

// 3. You CAN'T use """ inside text block without escaping
String html = """
        She said \"""Hello\"""
        """; // Escape one or more of the quotes

// 4. String methods work on text blocks
String trimmed = """
        hello
        """.stripIndent(); // Already stripped, but stripIndent() available for dynamic strings
```

## 3. Records (Preview) — JEP 359

Records are immutable data carriers — they automatically generate `equals()`, `hashCode()`, `toString()`, and accessor methods. Final in Java 16.

```java
// Before Records — a simple DTO required enormous boilerplate:
public final class Money {
    private final BigDecimal amount;
    private final String currency;

    public Money(BigDecimal amount, String currency) {
        this.amount = Objects.requireNonNull(amount);
        this.currency = Objects.requireNonNull(currency);
    }

    public BigDecimal amount() { return amount; }
    public String currency() { return currency; }

    @Override public boolean equals(Object o) { ... } // 10+ lines
    @Override public int hashCode() { return Objects.hash(amount, currency); }
    @Override public String toString() { return "Money[amount=" + amount + ", currency=" + currency + "]"; }
}

// With Records — 1 line!
public record Money(BigDecimal amount, String currency) {}

// Usage
Money price = new Money(new BigDecimal("29.99"), "GBP");
System.out.println(price.amount());    // 29.99
System.out.println(price.currency());  // GBP
System.out.println(price);            // Money[amount=29.99, currency=GBP]
```

### Real-World: Domain Model with Records

```java
// Value objects — perfect for Records
public record CustomerId(UUID value) {
    // Compact constructor for validation
    public CustomerId {
        Objects.requireNonNull(value, "Customer ID cannot be null");
    }

    public static CustomerId generate() {
        return new CustomerId(UUID.randomUUID());
    }

    public static CustomerId of(String uuid) {
        return new CustomerId(UUID.fromString(uuid));
    }
}

public record Money(BigDecimal amount, Currency currency) {
    public Money {
        Objects.requireNonNull(amount, "Amount required");
        Objects.requireNonNull(currency, "Currency required");
        if (amount.compareTo(BigDecimal.ZERO) < 0) {
            throw new IllegalArgumentException("Amount cannot be negative: " + amount);
        }
        // Normalise to 2 decimal places
        amount = amount.setScale(2, RoundingMode.HALF_UP);
    }

    public Money add(Money other) {
        if (!this.currency.equals(other.currency)) {
            throw new CurrencyMismatchException(this.currency, other.currency);
        }
        return new Money(this.amount.add(other.amount), this.currency);
    }

    public Money multiply(BigDecimal factor) {
        return new Money(this.amount.multiply(factor), this.currency);
    }
}

public record OrderSummary(
    String orderId,
    String customerEmail,
    Money total,
    int itemCount,
    OrderStatus status,
    LocalDateTime createdAt
) {}

// DTOs — replace Lombok @Data
public record CreateOrderRequest(
    String customerId,
    List<OrderItem> items,
    String deliveryAddress
) {}

public record OrderResponse(
    String id,
    String status,
    BigDecimal total,
    String currency,
    String message
) {
    // Additional constructor
    public static OrderResponse success(Order order) {
        return new OrderResponse(
            order.getId(),
            order.getStatus().name(),
            order.getTotal().amount(),
            order.getTotal().currency().getCurrencyCode(),
            "Order placed successfully"
        );
    }
}

// Event objects — immutable by nature
public record OrderPlacedEvent(
    String orderId,
    String customerId,
    Money amount,
    Instant occurredAt
) implements DomainEvent {
    public OrderPlacedEvent(String orderId, String customerId, Money amount) {
        this(orderId, customerId, amount, Instant.now());
    }
}
```

### Records and Interfaces

```java
// Records can implement interfaces
public interface Auditable {
    Instant createdAt();
    String createdBy();
}

public record AuditedOrder(
    String id,
    Money amount,
    Instant createdAt,
    String createdBy
) implements Auditable {}

// Records can be generic
public record Page<T>(
    List<T> content,
    int pageNumber,
    int pageSize,
    long totalElements
) {
    public int totalPages() {
        return (int) Math.ceil((double) totalElements / pageSize);
    }

    public boolean hasNext() {
        return pageNumber < totalPages() - 1;
    }
}

// Usage
Page<OrderSummary> page = new Page<>(orders, 0, 20, 150L);
```

### Corner Cases and Gotchas

```java
// 1. Records are implicitly final — cannot be extended
public record Point(int x, int y) {}
public class Point3D extends Point {} // COMPILE ERROR

// 2. Record components are final — cannot be mutated
var p = new Point(1, 2);
p.x = 3; // COMPILE ERROR — no setters, fields are final

// 3. Compact constructor — parameters are mutable, assignment is automatic AFTER
public record Range(int min, int max) {
    public Range {
        if (min > max) {
            // Can reassign parameters before implicit assignment
            int temp = min;
            min = max;
            max = temp;
        }
        // Implicit: this.min = min; this.max = max; happens after compact constructor
    }
}

// 4. Records cannot have instance fields (only record components)
public record Order(String id) {
    private String cached; // COMPILE ERROR — non-static field not allowed
    private static final String PREFIX = "ORD"; // OK — static fields allowed
}

// 5. Records CAN have additional methods
public record Circle(double radius) {
    public double area() { return Math.PI * radius * radius; }
    public double circumference() { return 2 * Math.PI * radius; }
}

// 6. Serialisation — records implement Serializable differently
// The serialised form is based on components, not fields
// Adding a component is a breaking serialisation change
```

## 4. Helpful NullPointerExceptions — JEP 358

**One of the most practically useful Java 14 improvements for debugging.**

```java
// Before Java 14 — cryptic NPE
// Exception in thread "main" java.lang.NullPointerException
//     at OrderService.processOrder(OrderService.java:42)
// Which of the 5 null-related things on that line caused it?

// Java 14 — precise NPE message
Customer customer = null;
String city = customer.getAddress().getCity().getName();
// Exception: Cannot invoke "Customer.getAddress()" because "customer" is null

String city = customer.getAddress().getCity().getName();
// Exception: Cannot invoke "City.getName()" because the return value of
// "Address.getCity()" is null
```

### Real-World Impact

```java
// Complex nested access — old NPE told you nothing useful
public class OrderProcessor {

    public String getDeliveryPostcode(Order order) {
        return order.getCustomer()
                    .getAddress()
                    .getPostcode()
                    .toUpperCase();
    }
}

// Java 14 NPE messages now tell you exactly what was null:
// "Cannot invoke 'Address.getPostcode()' because the return value of
//  'Customer.getAddress()' is null"

// Array access NPE is also improved
int[] arr = null;
int val = arr[0];
// "Cannot load from int array because 'arr' is null"

int[][] matrix = new int[3][];
int val = matrix[0][1];
// "Cannot load from int array because 'matrix[0]' is null"
```

> 💡 **Note:** Helpful NPEs are enabled by default from Java 14 onwards. In Java 14–17 they could be disabled with `-XX:-ShowCodeDetailsInExceptionMessages`. From Java 21 always on.

## 5. Pattern Matching for instanceof (Preview) — JEP 305

Eliminates the redundant cast after `instanceof`. Final in Java 16.

```java
// Before — verbose and error-prone double cast
if (obj instanceof String) {
    String s = (String) obj;  // Redundant — we KNOW it's a String
    System.out.println(s.toUpperCase());
}

// Java 14 Preview / Java 16 Final
if (obj instanceof String s) {
    System.out.println(s.toUpperCase()); // s is scoped to the if block
}

// Can combine with conditions
if (obj instanceof String s && s.length() > 5) {
    System.out.println("Long string: " + s);
}
```

### Real-World: Event Handling

```java
public class DomainEventHandler {

    public void handle(DomainEvent event) {
        // Before — instanceof chains with casts
        if (event instanceof OrderPlacedEvent) {
            OrderPlacedEvent e = (OrderPlacedEvent) event;
            notifyCustomer(e.getCustomerId(), e.getOrderId());
            updateInventory(e.getItems());
        } else if (event instanceof OrderCancelledEvent) {
            OrderCancelledEvent e = (OrderCancelledEvent) event;
            refundPayment(e.getPaymentId());
            releaseInventory(e.getItems());
        }

        // Java 16+ — clean, no redundant casts
        if (event instanceof OrderPlacedEvent e) {
            notifyCustomer(e.customerId(), e.orderId());
            updateInventory(e.items());
        } else if (event instanceof OrderCancelledEvent e) {
            refundPayment(e.paymentId());
            releaseInventory(e.items());
        } else if (event instanceof PaymentFailedEvent e && e.isRetryable()) {
            scheduleRetry(e.paymentId(), e.retryAfter());
        }
    }
}
```

## 6. `Stream.toList()` and Other Collector Improvements

```java
// Java 16 — Stream.toList() shorthand
// Before:
List<String> names = stream.collect(Collectors.toList());

// Java 16+:
List<String> names = stream.toList(); // Unmodifiable list — unlike Collectors.toList()

// IMPORTANT DIFFERENCE:
List<String> mutable = stream.collect(Collectors.toList()); // Mutable
List<String> immutable = stream.toList();                   // IMMUTABLE — throws on add()

// Collectors.teeing() — Java 12
// Collect into two collectors simultaneously, then merge results
record MinMax(Optional<Integer> min, Optional<Integer> max) {}

MinMax result = Stream.of(3, 1, 4, 1, 5, 9, 2, 6)
    .collect(Collectors.teeing(
        Collectors.minBy(Comparator.naturalOrder()),
        Collectors.maxBy(Comparator.naturalOrder()),
        (min, max) -> new MinMax(min, max)
    ));
// MinMax[min=Optional[1], max=Optional[9]]

// Real-world: compute count and sum in one pass
record Stats(long count, BigDecimal sum) {}
Stats stats = orders.stream()
    .collect(Collectors.teeing(
        Collectors.counting(),
        Collectors.reducing(BigDecimal.ZERO, Order::getAmount, BigDecimal::add),
        (count, sum) -> new Stats(count, sum)
    ));
```
