## 1. Text Blocks (Final) — JEP 378

Text blocks become standard in Java 15. See the preview notes in [Java 12–14](./java-12-14.md) for full details. Key additions that became final:

```java
// String template methods added for text blocks:

// formatted() — same as String.format() but as instance method
String payload = """
        {
          "name": "%s",

        """.formatted(customer.getName(), amount, currency);

// stripIndent() — removes incidental whitespace from a regular string

String stripped = raw.stripIndent(); // "hello\nworld\n"

// translateEscapes() — interprets escape sequences in regular strings
String s = "line1\\nline2".translateEscapes(); // "line1\nline2"
```

## 2. Sealed Classes (Preview) — JEP 360

### What It Is

Sealed classes and interfaces **restrict which classes can extend or implement them**. This enables **closed type hierarchies** — you define exactly which subtypes exist.

This is a fundamental improvement for domain modelling — you can represent a finite set of shapes (like payment methods or domain events) with compile-time exhaustiveness checking.

### Syntax

```java
// Sealed class — only permits listed classes to extend it
public sealed class Shape permits Circle, Rectangle, Triangle {
    public abstract double area();
}

// Each permitted class must be:
// 1. final (no further extension)
// 2. sealed (further restricted)
// 3. non-sealed (open to any extension)

public final class Circle extends Shape {
    private final double radius;
    public Circle(double radius) { this.radius = radius; }
    public double area() { return Math.PI * radius * radius; }
}

public final class Rectangle extends Shape {
    private final double width, height;
    public Rectangle(double width, double height) {
        this.width = width; this.height = height;
    }
    public double area() { return width * height; }
}

public non-sealed class Triangle extends Shape {
    // non-sealed — anyone can extend Triangle
    private final double base, height;
    public Triangle(double base, double height) {
        this.base = base; this.height = height;
    }
    public double area() { return 0.5 * base * height; }
}
```

### Real-World: Payment Method Domain Model

```java
// Sealed hierarchy — exactly these payment methods exist in our system
public sealed interface PaymentMethod
    permits CreditCard, DebitCard, BankTransfer, Cryptocurrency, PayPal {

    Money getMaxAmount();
    String getDisplayName();

}

    public Money getMaxAmount() { return Money.of(50000, "GBP"); }

}
```

### Real-World: Result Type (like Rust's Result<T, E>)

```java
// A common pattern — sealed Result type
public sealed interface Result<T>
    permits Result.Success, Result.Failure {

    record Success<T>(T value) implements Result<T> {}
    record Failure<T>(String errorCode, String message, Exception cause)
        implements Result<T> {
        public Failure(String errorCode, String message) {
            this(errorCode, message, null);
        }
    }

    static <T> Result<T> success(T value) { return new Success<>(value); }
    static <T> Result<T> failure(String code, String message) {
        return new Failure<>(code, message);
    }

    default boolean isSuccess() { return this instanceof Success<T>; }

    default T getOrThrow() {
        return switch (this) {
            case Success<T> s -> s.value();
            case Failure<T> f -> throw new BusinessException(f.errorCode(), f.message());
        };
    }

    default <R> Result<R> map(Function<T, R> mapper) {
        return switch (this) {
            case Success<T> s -> Result.success(mapper.apply(s.value()));
            case Failure<T> f -> new Failure<>(f.errorCode(), f.message(), f.cause());
        };
    }
}

// Usage
public Result<Order> placeOrder(OrderRequest request) {
    if (!inventoryService.isAvailable(request.items())) {
        return Result.failure("INVENTORY_UNAVAILABLE", "Items out of stock");
    }

    try {
        Order order = orderRepository.save(createOrder(request));
        return Result.success(order);
    } catch (Exception e) {
        return new Result.Failure<>("ORDER_CREATION_FAILED", e.getMessage(), e);
    }
}

// At the call site:
Result<Order> result = orderService.placeOrder(request);
switch (result) {
    case Result.Success<Order> s  -> sendConfirmation(s.value());
    case Result.Failure<Order> f  -> sendErrorResponse(f.errorCode(), f.message());
}
```

### Real-World: Domain Events with Sealed Interfaces

```java
public sealed interface OrderEvent
    permits OrderPlaced, OrderConfirmed, OrderShipped, OrderDelivered,
            OrderCancelled, OrderRefunded {
    String orderId();
    Instant occurredAt();
}

public record OrderPlaced(String orderId, String customerId, Money amount, Instant occurredAt)
    implements OrderEvent {}

public record OrderCancelled(String orderId, String reason, Instant occurredAt)
    implements OrderEvent {}

public record OrderShipped(String orderId, String trackingNumber, Instant occurredAt)
    implements OrderEvent {}

// Event handler — exhaustiveness guaranteed by compiler
public class OrderEventProjection {

    public void apply(OrderEvent event) {
        switch (event) {
            case OrderPlaced e      -> createOrderRecord(e);
            case OrderConfirmed e   -> updateStatus(e.orderId(), "CONFIRMED");
            case OrderShipped e     -> updateShipping(e.orderId(), e.trackingNumber());
            case OrderDelivered e   -> markDelivered(e.orderId(), e.occurredAt());
            case OrderCancelled e   -> cancelOrder(e.orderId(), e.reason());
            case OrderRefunded e    -> processRefund(e.orderId());
            // If you add a new OrderEvent subtype and forget to handle it here,
            // the compiler tells you immediately — no runtime surprises
        }
    }
}
```

### Corner Cases and Gotchas

```java
// 1. Permitted classes must be in same package or same compilation unit
// com.example.payment.PaymentMethod must permit classes in com.example.payment
// Cannot permit a class in com.example.billing from a different package structure

// 2. All permitted classes must explicitly extend/implement the sealed type
sealed interface Vehicle permits Car, Truck {}
// Car MUST implement Vehicle, or compilation fails
// You cannot have an unused entry in permits

// 3. Sealed + records = perfect pairing (records are implicitly final)
public sealed interface Command permits CreateOrder, CancelOrder {}
public record CreateOrder(String customerId, List<Item> items) implements Command {}
public record CancelOrder(String orderId, String reason) implements Command {}
// No need to write 'final' on records — they already are

// 4. Sealed classes in switch — exhaustiveness without default
Shape shape = getShape();
double area = switch (shape) {
    case Circle c       -> Math.PI * c.radius() * c.radius();
    case Rectangle r    -> r.width() * r.height();
    case Triangle t     -> 0.5 * t.base() * t.height();
    // No default needed — compiler knows all subtypes
};
// If Triangle is non-sealed, switch MAY still need default
// depending on whether further subtypes exist
```

## 3. Records (Final) — JEP 395

Records become a permanent Java feature. See the preview in [Java 12–14](./java-12-14.md) for full details. Key finalised behaviours:

```java
// Records and Jackson (JSON serialisation) — a common real-world pattern
// Jackson 2.12+ supports records natively
@JsonIgnoreProperties(ignoreUnknown = true)
public record CustomerDTO(
    @JsonProperty("id")   String id,
    @JsonProperty("name") String name,
    @JsonProperty("email") String email
) {}

// Records in JPA — NOT directly usable as @Entity (entities need mutability)
// Use records as DTOs / projections:
public interface OrderProjection {
    record OrderSummary(String id, String customerName, BigDecimal total) {}

    @Query("SELECT new com.example.OrderProjection.OrderSummary(o.id, c.name, o.total) " +
           "FROM Order o JOIN o.customer c WHERE o.status = :status")
    List<OrderSummary> findSummariesByStatus(OrderStatus status);
}

// Records as Map keys — equals/hashCode are component-based
Map<Money, String> labels = new HashMap<>();
labels.put(new Money(BigDecimal.TEN, "GBP"), "Ten pounds");
// new Money(BigDecimal.TEN, "GBP").equals(new Money(BigDecimal.TEN, "GBP")) = true
// Safe to use as map keys — unlike mutable objects
```

## 4. Pattern Matching for instanceof (Final) — JEP 394

Becomes a permanent feature in Java 16. See preview in [Java 12–14](./java-12-14.md).

```java
// Negation pattern — note the scope
if (!(obj instanceof String s)) {
    return; // s is NOT in scope here
}
// s IS in scope here — because we know it's a String past this point
processString(s);

// Used in equals() — the classic pattern
public record Point(int x, int y) {
    @Override
    public boolean equals(Object o) {
        return o instanceof Point p && p.x == x && p.y == y;
        // Replaces: if (!(o instanceof Point)) return false; Point p = (Point) o; ...
    }
}
```

## 5. Sealed Classes (Final) — JEP 409

Sealed classes become permanent in Java 17. Full details above in the Java 15 section.

```java
// Java 17 — the full combination: sealed + records + pattern matching
// This trio is the modern Java domain modelling toolkit

public sealed interface PaymentResult
    permits PaymentResult.Approved, PaymentResult.Declined, PaymentResult.Error {

    record Approved(String transactionId, Money amount, Instant processedAt)
        implements PaymentResult {}

    record Declined(String reason, String code)
        implements PaymentResult {}

    record Error(String message, boolean isRetryable)
        implements PaymentResult {}
}

// Usage — exhaustive switch, no casts, no null checks on type
public ResponseEntity<PaymentResponse> processPayment(PaymentRequest req) {
    PaymentResult result = paymentGateway.process(req);

    return switch (result) {
        case PaymentResult.Approved a -> ResponseEntity.ok(
            new PaymentResponse("SUCCESS", a.transactionId(), a.amount())
        );
        case PaymentResult.Declined d -> ResponseEntity.status(402).body(
            new PaymentResponse("DECLINED", null, null)
        );
        case PaymentResult.Error e when e.isRetryable() -> ResponseEntity.status(503).body(
            new PaymentResponse("RETRY", null, null)
        );
        case PaymentResult.Error e -> ResponseEntity.status(500).body(
            new PaymentResponse("ERROR", null, null)
        );
    };
}
```

## 6. Strong Encapsulation of JDK Internals — JEP 403

Java 17 makes strong encapsulation of JDK internals permanent. Internal APIs (`sun.*`, `com.sun.*`) are now inaccessible by default.

```bash
# Previously allowed (Java 9–16 with warnings, Java 17+ forbidden):
# sun.misc.Unsafe, sun.reflect.*, com.sun.* packages

# Java 17 — accessing internals throws InaccessibleObjectException
# Many libraries that relied on internal APIs needed to be updated

# Check your dependencies:
java --illegal-access=warn -jar myapp.jar  # No longer works in Java 17

# The --add-opens workaround (last resort, use modern library versions instead):
java --add-opens java.base/java.lang=ALL-UNNAMED -jar myapp.jar
```

### Migration Impact

| Library | Fix Needed | Solution |
|---------|-----------|---------|
| Spring Framework < 5.3 | Yes | Upgrade to Spring 5.3+ / Spring Boot 2.5+ |
| Hibernate < 5.6 | Yes | Upgrade to Hibernate 5.6+ / 6+ |
| Jackson < 2.12 | Partial | Upgrade to 2.12+ |
| Mockito < 4.0 | Yes | Upgrade to Mockito 4+ |
| PowerMock | Problematic | Migrate away — use Mockito directly |
| Lombok < 1.18.20 | Yes | Upgrade to 1.18.20+ |

## 7. Enhanced Pseudo-Random Number Generators — JEP 356

```java
// New RandomGenerator interface — consistent API for all RNG implementations
RandomGenerator rng = RandomGeneratorFactory.of("Xoshiro256PlusPlus").create();

// Cryptographically secure
RandomGenerator secure = new SecureRandom();

// Stream of random numbers
rng.ints(10, 1, 100).forEach(System.out::println);
rng.doubles(5, 0.0, 1.0).forEach(System.out::println);

// Real-world: A/B test bucket assignment
public class ABTestService {
    private final RandomGenerator rng = RandomGeneratorFactory
        .of("Xoshiro256PlusPlus").create();

    public String assignBucket(String userId, String experiment) {
        // Deterministic based on userId + experiment name
        long seed = (userId + experiment).hashCode();
        var deterministicRng = RandomGeneratorFactory
            .of("Xoshiro256PlusPlus").create(seed);
        return deterministicRng.nextDouble() < 0.5 ? "control" : "treatment";
    }
}
```

## 8. `instanceof` Pattern Variables in Complex Conditions — Java 16/17

```java
// Pattern variable scoping rules
public class TypeSafeProcessor {

    public void process(Object input) {
        // AND — both must be true; s is in scope in the AND condition and body
        if (input instanceof String s && !s.isBlank()) {
            handle(s.strip());
        }

        // OR — s is NOT in scope (either branch might fail)
        // if (input instanceof String s || condition) { use(s); } // COMPILE ERROR

        // Negation — s not in scope in the if body; IS in scope after return
        if (!(input instanceof String s)) {
            throw new IllegalArgumentException("Expected String");
        }
        // s is in scope here — compiler knows input IS a String
        process(s);
    }

    // Practical: JSON node processing
    public BigDecimal extractAmount(Object jsonNode) {
        if (jsonNode instanceof Map<?, ?> map
                && map.get("amount") instanceof Number n) {
            return new BigDecimal(n.toString());
        }
        throw new JsonParseException("Cannot extract amount from: " + jsonNode);
    }
}
```

## 9. New Vector API (Incubator) — JEP 338 / 414

Enables SIMD (Single Instruction Multiple Data) vector operations — important for ML, image processing, financial calculations.

```java
// Incubator in Java 16/17 — requires --add-modules jdk.incubator.vector
import jdk.incubator.vector.*;

// SIMD vector addition — processes multiple values in one CPU instruction
static final VectorSpecies<Float> SPECIES = FloatVector.SPECIES_256; // 8 floats at once

public float[] vectorAdd(float[] a, float[] b) {
    float[] result = new float[a.length];
    int i = 0;

    // Process 8 floats at a time with SIMD
    for (; i < SPECIES.loopBound(a.length); i += SPECIES.length()) {
        var va = FloatVector.fromArray(SPECIES, a, i);
        var vb = FloatVector.fromArray(SPECIES, b, i);
        va.add(vb).intoArray(result, i);
    }

    // Handle remainder
    for (; i < a.length; i++) {
        result[i] = a[i] + b[i];
    }
    return result;
}
```

## 10. Context-Specific Deserialization Filters — JEP 415

Protects against deserialization attacks by filtering incoming object streams.

```java
// Global filter — set at JVM startup
ObjectInputFilter.Config.setSerialFilter(info -> {
    Class<?> cl = info.serialClass();
    if (cl == null) return ObjectInputFilter.Status.ALLOWED;

    // Only allow your known safe classes
    if (cl.getName().startsWith("com.mycompany.")) {
        return ObjectInputFilter.Status.ALLOWED;
    }
    // Block everything else
    return ObjectInputFilter.Status.REJECTED;
});

// Per-stream filter — fine-grained control
ObjectInputStream ois = new ObjectInputStream(inputStream);
ois.setObjectInputFilter(info -> {
    if (info.depth() > 5) return ObjectInputFilter.Status.REJECTED; // Too deep
    if (info.references() > 100) return ObjectInputFilter.Status.REJECTED; // Too many refs
    return ObjectInputFilter.Status.UNDECIDED; // Defer to global filter
});
```
